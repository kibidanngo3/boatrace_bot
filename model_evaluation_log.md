# モデル評価ログ

各モデル変更の効果を切り分けて比較するための基準記録。変更は1つずつ行い、
同じホールドアウト期間・同じ条件で評価する。

## 共通の評価条件

- 学習データ: `training_data.csv`（mbrace.or.jpの公式B/K配信データ、`scripts/build_training_data_v2.py`で収集）
- 学習期間: 2025-07-11 〜 2026-05-11
- ホールドアウト期間（検証専用・学習に不使用）: 2026-05-12 〜 2026-07-10（60日）
- 特徴量: `scripts/train_model.py` の `FEATURES`（wind_speed, wave, is_debuff_1, rank_val_i, win_rate_i, ex_time_i, ex_diff_i, ex_rank_i, st_i）
- 賭け戦略トリガー: `IN_JUMP_THRESHOLD=0.55`, `FOCUS_TOP_THRESHOLD=0.35`, `STANDARD_TOP_THRESHOLD=0.25`
- EV閾値: `MIN_EXPECTED_VALUE=1.00`
- ステーク: 1/4ケリー（`KELLY_FRACTION=0.25`）、バックテストでは1レースごとにバンクロール10,000円固定という単純化（累積シミュレーションではない）

- Accuracy（全6艇・argmax、検証全件 n=8544）: **61.3%**
- LogLoss（検証全件）: **1.0678**
- 較正チェック（イン飛び予測局面、in_jump_prob>=0.55 の検証データ n=2774）:
  モデルの平均予測1号艇勝率 **26.6%** / 実際の1号艇勝率 **26.3%**（ほぼ一致）
- 賭け対象局面での1着的中率（top1艇が実際に1着になった率、検証全件ベース n=2512）:

### まだ計測していないもの（既知のギャップ）

- **最大ドローダウン**: 未計測。上記バックテストは1レースごとバンクロール固定の簡易シミュレーションで、
  時系列で資金がどう推移したか（連敗時の資金減少幅など）は追っていない。累積シミュレーションでの再計測が必要。
- Brier Score / Expected Calibration Error: 未計測（イン飛び局面の較正チェックのみ実施）。
- 期間分割（前半/後半30日）・会場別・オッズ帯別の安定性: 未計測。
- 着順確率（2着・3着）は現状Plackett-Luce近似（実測データからの直接学習ではない）。

## 2着・3着専用モデル（order_model_2nd/3rd_v1）

学習: `scripts/train_order_model.py`。特徴量は`train_model.py`のFEATURESに加えて、
2着モデルは実際の1着艇番(`given_1st`)、3着モデルは実際の1・2着艇番(`given_1st`,`given_2nd`)を
追加入力とし、実測の2着・3着ラベルから直接学習（検証60日ホールドアウトは同一）。

### Plackett-Luce近似との比較（1着確率ベクトルの再正規化 vs 専用モデル、検証全件 n=8544）

| | Accuracy | LogLoss |
|---|---:|---:|
| 2着 PL近似 | 42.8% | 1.3853 |
| 2着 専用モデル | **44.7%** | **1.3014** |
| 3着 PL近似 | 40.2% | 1.4005 |
| 3着 専用モデル | **41.7%** | **1.2329** |

### 実オッズ・実結果バックテスト（ホールドアウト期間、n=1783サンプル、v5基準値と同一条件）

| | v5のみ(PL近似) | v5+着順モデル |
|---|---:|---:|
| 賭け成立数 | 980 | 490 |
| 的中率 | 7.0% | **11.4%** |
| 投資額 | 203,800円 | 104,700円 |
| 払戻額 | 710,670円 | 452,520円 |
| 回収率 | 348.7% | **432.2%** |
| FOCUS回収率 | 372.0% | **433.8%** |
| STANDARD回収率 | 155.9% | **416.2%** |

賭け成立数はほぼ半減（EVフィルターが過大評価されたチケットをより多く弾くようになったため）だが、
的中率・回収率は明確に改善。上位5件の的中を除いても回収率約309%を維持しており、一部の高配当的中に
依存した結果ではない。→ **本番デプロイ推奨**（ユーザー確認待ち）。
